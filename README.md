## What this project does

Clara Answers provides AI voice agents for service trade companies — fire protection, HVAC, alarm systems, electrical contractors, sprinkler companies.

Every new client goes through two calls:

- **Demo call** — client describes their operations. Details are often incomplete or vague.
- **Onboarding call** — everything gets confirmed, finalized, and locked in.

This pipeline automates what happens after each call:

```
Demo transcript
  → LLM extracts structured operational data
  → Pydantic validates the schema (retries if invalid)
  → Generates Account Memo v1
  → Generates Agent Spec v1 (Retell-style voice agent config)

Onboarding transcript + memo v1
  → LLM patches and updates the data
  → Pydantic validates again
  → Generates Account Memo v2
  → Generates Agent Spec v2
  → Generates Changelog (field-level diff v1 → v2)
```

---

### Pipeline A — Demo call → Preliminary agent ✅
- Extracts structured account memo from demo transcript
- Generates Retell-style agent spec with full system prompt
- Saves versioned outputs under `outputs/accounts/<account_id>/v1/`
- Logs each run to `outputs/run_log.json` (task tracker)

### Pipeline B — Onboarding → Agent update ✅
- Loads v1 memo and patches it with onboarding data
- Preserves all confirmed v1 fields, updates only what changed
- Generates updated memo v2 and agent spec v2
- Produces `changelog.json` with field-level diff (DeepDiff)
- Saves under `outputs/accounts/<account_id>/v2/`

### Versioning and diff ✅
- v1 generated from demo transcript only
- v2 generated by patching v1 with onboarding data
- `changelog.json` shows every field that was added, updated, or removed
- No fields are silently overwritten

### n8n orchestration ✅
- Two workflow JSON exports in `workflows/`
- Demo pipeline: Manual Trigger → Read Transcript → Extract → Validate → Save
- Onboarding pipeline: Manual Trigger → Check v1 exists → Extract → Validate → Diff → Save
- Visual workflow canvas demonstrating orchestration design

## Data flow

```
dataset/
  demo_calls/           5 demo transcripts
  onboarding_calls/     5 onboarding transcripts
        │
        ▼
scripts/
  extract_demo.py       Stage 1 — Groq LLM extraction + Pydantic validation
  extract_onboarding.py Stage 2 — patch v1, validate, generate diff
  generate_agent.py     Build Retell-style agent spec + system prompt
  generate_diff.py      DeepDiff changelog v1 → v2
  validate_schema.py    Pydantic schema validation with retry
  task_tracker.py       Log every run to run_log.json
        │
        ▼
outputs/
  run_log.json          Task tracker — all pipeline runs
  accounts/
    ACC-001/
      v1/ memo.json + agent_spec.json
      v2/ memo.json + agent_spec.json + changelog.json
```

## Setup

**1. Get a free Groq API key**

Sign up at [console.groq.com](https://console.groq.com) → API Keys → Create API Key

**2. Configure environment**

```bash
cp .env.example .env
# Open .env and add: GROQ_API_KEY=your_key_here
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

**4. Run the UI**

```bash
streamlit run app.py
```

Opens at **http://localhost:8501**

## Running the pipeline

### Via Streamlit UI

1. Open **▶️ Run Pipeline** tab
2. Select an account (e.g. `ACC-001 — AquaGuard Fire Protection`)
3. Click **Run Demo Pipeline** → wait for ✅
4. Click **Run Onboarding Pipeline** → wait for ✅
5. Check **📄 View Outputs** and **🔄 Changelog** tabs

Or click **Run All Accounts** to batch process all 5 at once.

### Via CLI

```bash
cd scripts

# Single account
python extract_demo.py --account ACC-001 --transcript ../dataset/demo_calls/demo_001_aquaguard.txt
python extract_onboarding.py --account ACC-001 --transcript ../dataset/onboarding_calls/onboarding_001_aquaguard.txt

# All 5 accounts
python run_all.py
```

## n8n workflow setup (optional)

```bash
sudo systemctl start docker    # Linux only
docker compose up -d
```

Open **http://localhost:5678**, create your account on first launch, then:

1. Workflows → Add workflow → Import from file
2. Import `workflows/demo_pipeline.json`
3. Import `workflows/onboarding_pipeline.json`
4. Open a workflow → Set Parameters node → set `account_id` and `transcript_path`
5. Click Execute Workflow

> Paths inside Docker use `/data/` prefix e.g. `/data/dataset/demo_calls/demo_001_aquaguard.txt`

## Retell integration note

The `agent_spec.json` is formatted to match the Retell agent configuration structure. Because programmatic agent creation requires a paid Retell plan, the recommended approach is:

1. Open your Retell dashboard
2. Create a new agent
3. Copy the `system_prompt` field from `agent_spec.json` into the agent prompt
4. Set the transfer number and fallback from `transfer_protocol`

---

## Known limitations

- **Audio not supported** — pipeline accepts text transcripts only. For audio, run Whisper locally first.
- **No live Retell API** — agent spec is Retell-compatible JSON but requires manual import. Programmatic creation needs a paid plan.
- **Local task tracker** — runs logged to `run_log.json` instead of Asana. Production version would push to Asana 
- **English only** — Spanish-speaking caller requirements are flagged in `questions_or_unknowns` but not handled in the prompt.
